# MikroTik Exploit Scanner v3.0 Proxy Checker

**MikroTik Exploit Scanner v3.0 Proxy Checker** — это программа для сканирования уязвимостей устройств MikroTik RouterOS через протокол Winbox (CVE-2018-14847) и проверки прокси-серверов. Программа позволяет сканировать подсети IPv4, извлекать логины и пароли RouterOS, а также проверять прокси (HTTP, SOCKS4, SOCKS5).

⚠️ **Внимание**: Этот инструмент предназначен исключительно для образовательных целей и тестирования собственных сетей. Использование программы без согласия владельца оборудования запрещено и может нарушать законы вашей страны.

## Описание

### Сканирование уязвимостей MikroTik
Программа позволяет сканировать устройства MikroTik на уязвимость CVE-2018-14847 через протокол Winbox (по умолчанию используется порт 8291). Вы можете сканировать:
- Один IP-адрес (например, 192.168.1.10).
- Диапазон IP-адресов (например, 192.168.1.0/24).
- Список IP-адресов из файла.

Результаты сканирования отображаются в окне программы и сохраняются в файл `results.txt`. Для каждого уязвимого устройства выводится:
- IP-адрес.
- Версия RouterOS.
- Геолокация (страна, ASN, организация).
- Логин и пароль (если удалось извлечь).

⚠️ **Важно**: Уязвимость работает только на версиях RouterOS до 6.45.9. На более новых версиях она не работает.

### Проверка прокси-серверов
Программа также позволяет проверять прокси-серверы:
- Поддерживаются типы: HTTP, SOCKS4, SOCKS5.
- Можно проверить один прокси или список прокси из файла.
- Результаты проверки сохраняются в файлы `valid_proxies.txt` (валидные прокси) и `invalid_proxies.txt` (невалидные прокси).
- Для каждого прокси отображается пинг (в миллисекундах).

### Защита маршрутизатора MikroTik
Чтобы защитить своё устройство MikroTik от данной уязвимости:
1. Обновите RouterOS до версии 6.45.9 или выше.
2. Ограничьте доступ к Winbox через брандмауэр:
   - Войдите в настройки IP-фильтра брандмауэра.
   - Добавьте правило:  
     ```
     chain=input in-interface=wan protocol=tcp dst-port=8291 action=drop
     ```

## Сборка из исходников

### Установка инструментов (Windows)
Для работы программы вам нужно установить несколько инструментов:

1. **Git**  
   Скачайте и установите Git для клонирования репозитория:  
   [Скачать Git](https://git-scm.com/download/win)

2. **Python**  
   Установите Python версии 3.6 или выше:  
   [Скачать Python](https://www.python.org/downloads/)  
   Во время установки не забудьте отметить опцию "Add Python to PATH".

3. **PowerShell** (опционально)  
   PowerShell нужен для удобной работы с командами в Windows. Если у вас Windows 10/11, он уже установлен. Если нет, скачайте:  
   [Скачать PowerShell](https://github.com/PowerShell/PowerShell/releases/)

### Установка программы
1. **Склонируйте репозиторий**:
   Откройте PowerShell (или командную строку) и выполните:
   ```
   git clone https://github.com/nebbbes/Mikrotik_Checker
   cd Mikrotik_Checker
   ```

2. **Установите зависимости**:
   Программа требует несколько библиотек Python. Установите их с помощью команды:
   ```
   pip install -r requirements.txt
   ```

3. **Запустите программу**:
   Запустите графический интерфейс программы:
   ```
   python src/gui.py
   ```

### Сборка исполняемого файла (опционально)
Если вы хотите создать исполняемый файл (например, `proga.exe` для Windows):
1. Установите PyInstaller:
   ```
   pip install pyinstaller
   ```
2. Выполните сборку:
   ```
   pyinstaller --onefile --noconsole --add-data "fon.png;." --add-data "icon.png;." --name proga gui.py
   ```
3. Исполняемый файл появится в папке `dist/`. Запустите его двойным кликом.

## Использование

### Сканирование уязвимостей
1. **Запустите программу**:
   - Если вы используете исходный код: `python src/gui.py`.
   - Если вы собрали исполняемый файл: запустите `proga.exe`.

2. **Выберите режим сканирования**:
   - **IP/CIDR**: Введите один IP-адрес (например, `192.168.1.10`) или диапазон (например, `192.168.1.0/24`).
   - **Файл**: Выберите файл со списком IP-адресов (нажмите "Обзор" и выберите файл).

3. **Настройте параметры**:
   - Укажите порт (по умолчанию 8291).
   - Выберите количество потоков сканирования (по умолчанию 10).

4. **Запустите сканирование**:
   - Нажмите кнопку "Старт".
   - Во время сканирования можно нажать "Пауза" для приостановки или "Стоп" для полной остановки.
   - Результаты отображаются в окне программы и сохраняются в `results.txt`.

5. **Фильтрация результатов**:
   - После сканирования можно отфильтровать результаты по стране или ASN с помощью выпадающих меню и кнопки "Применить фильтры".

6. **Просмотр статистики**:
   - Нажмите "Показать статистику", чтобы увидеть график с результатами сканирования.

### Проверка прокси-серверов
1. **Настройте прокси**:
   - В правой части интерфейса выберите тип прокси (HTTP, SOCKS4, SOCKS5 или "all" для проверки всех типов).
   - Введите адрес прокси в формате `ip:port` (например, `192.168.1.1:8080`).
   - (Опционально) Укажите логин и пароль, если прокси требует авторизации.
   - Укажите количество потоков проверки (по умолчанию 10).

2. **Проверка одного прокси**:
   - Нажмите "Тест прокси", чтобы проверить введённый прокси.
   - Результат (рабочий или нет, а также пинг) отобразится в окне программы.

3. **Проверка списка прокси**:
   - Нажмите "Тест списка прокси" и выберите файл со списком прокси (каждый прокси в формате `ip:port`, по одному на строку).
   - Во время проверки можно нажать "Пауза теста прокси" или "Отмена теста прокси".
   - Результаты сохраняются в файлы `valid_proxies.txt` (рабочие прокси) и `invalid_proxies.txt` (неработающие прокси).

## Горячие клавиши
- `Ctrl+S` — Начать сканирование.
- `Ctrl+P` — Пауза/Возобновить сканирование.
- `Ctrl+T` — Тестировать текущий прокси.
- `Ctrl+C` — Копировать текст из окна вывода.
- `Ctrl+Q` — Выход из приложения.

## Файлы
- `src/gui.py` — основной исходный код.
- `src/fon.png` — изображение для интерфейса.
- `src/icon.png` — иконка для системного трея.
- `requirements.txt` — список зависимостей.
- `results.txt` — результаты сканирования (создаётся автоматически).
- `success_ips.txt` — список уязвимых IP (создаётся автоматически).
- `valid_proxies.txt` — валидные прокси (создаётся после теста).
- `invalid_proxies.txt` — невалидные прокси (создаётся после теста).
- `config.ini` — файл конфигурации (создаётся автоматически при первом запуске).
